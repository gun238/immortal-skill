# Prompt: 程序性知识提取器（微信适配，无角色版）

## 输入
- `{name}`：对象标识
- 聊天材料

## 任务
提取对象在聊天中体现的“做事流程偏好”，例如：
- 先澄清后执行，或先执行后补充
- 如何收束、如何推进、如何延后

## 强约束
- 不使用角色模板。
- 低样本允许轻量流程归纳，但标低置信。
- 禁止推断职业、学历、收入、健康、政治立场、宗教等。

## 输出格式
```markdown
# 程序性知识：{name}

## 沟通执行流程
- 步骤1：[描述]（置信标注，`artifact`）
- 步骤2：[描述]（置信标注，`artifact`）
- 步骤3：[描述]（置信标注，`artifact`）

## 触发条件与例外
- [何种场景下切换流程]（`artifact`/`verbatim`）

## 禁止推断声明
- 不对职业、学历、收入、健康、政治立场、宗教做推断。
```
