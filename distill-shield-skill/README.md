## 防蒸馏是啥

你离职了，硬盘交给公司。

你立遗嘱了，数字资产给家人。

你合作了，项目文档发给合伙人。

你创作了，作品集打包发给平台。

**然后呢？**

对方把你交过去的东西——聊天记录、笔记、文档、作品——往自动化管线里一丢，RAG 一塞，LLM 一跑，你的说话方式、做事逻辑、甚至性格特征，全被抽成一个数字分身。

**你不知道、没授权、没法追。**

---

**防蒸馏做的事很具体：**

1. **Canary（金丝雀）**：在你的资料包里埋一串唯一标记。以后如果在不该出现的地方搜到了这串字——说明你的包**被原样或大段搬进了某个管线**。不是铁证，是线索，但有线索和没线索是两回事。

2. **可选策略段落**：在文档里加一些对机器不友好、对人可读性影响不大的内容。自动化管线吃进去会卡壳或出垃圾；人正常读不受影响。加多重、怎么加，看 [`SKILL.md`](SKILL.md) 里的 Phase。

**做不到的事（不吹牛）**：

- 加密到别人打不开——做不到。
- 保证全世界没有任何模型能学你——做不到。
- 替代律师函——做不到。

防蒸馏的价值是：**提高未授权蒸馏的成本 + 留下可追溯的痕迹 + 配合你的协议一起形成证据链**。够用了。

## 谁需要这个

- **要离职的你**：交给公司的文件夹，你想标明「别拿去训模型、别顺手蒸个我」。
- **立遗嘱的你**：数字遗产给继承人看，不是给某个 App 一键蒸馏的。
- **创作者**：作品集打包发出去，你想多一层「别蒸我风格」的标记。
- **任何要把自己的文字 / 记录交给别人的人**：交之前多想一步不亏。

**不该用的场景**：对别人的数据恶意投毒。这个工具只服务**你自己有权处置的材料**——不教、不支持、不讨论对他人的攻击。

## 能防什么不能防什么

| | 能帮你 | 别指望 |
|---|--------|--------|
| **技术** | Canary 追踪、策略段落干扰、清单审计 | 数学级别的「绝不可蒸」 |
| **法律** | 配合蒸馏协议形成证据意识 | 替代律师出庭 |
| **心理** | 「我说过不让你蒸」这句话有了载体 | 挡住铁了心抄的人 |

**军备竞赛永远存在**。盾的意义不是刀枪不入，是**让对方知道你不是没准备**。

## 怎么用

在仓库根目录执行：

```bash
python3 distill-shield-skill/kit/shield_gen.py --output ./handover-bundle --label "我的移交包"
```

跑完后 `./handover-bundle/` 里会有：

| 文件 | 干啥 |
|------|------|
| `CANARY.txt` | 那串唯一标记 + 一句人话说明 |
| `SHIELD-MANIFEST.json` | 结构化元数据（时间、标签、Canary 值） |

**然后**：把生成的文件合并进你真正要交的目录，按 [`SKILL.md`](SKILL.md) 做一遍自检就行。

## 和谁搭配

| 搭子 | 怎么配合 |
|------|----------|
| **[蒸馏协议](../distill-protocol-skill/README.md)** | 协议写规矩（能不能蒸、能不能商用），Shield 做技术标记——**又立字据又挂锁** |
| **[蒸笼 / 永生蒸馏器](../README.md)** | 蒸馏管「怎么从材料蒸出分身」，防蒸馏管「交出去之前怎么加固」——**两码事，别混** |

**推荐顺序**：先 **[蒸馏协议](../distill-protocol-skill/README.md)**（立规矩）→ 再 **防蒸馏**（加技术标记）→ 然后再决定要不要 **[蒸馏](../README.md)** 别人。
