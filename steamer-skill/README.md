## 蒸笼是啥

家里蒸馒头、包子、早点的那层竹笼，谁都见过。

我们取「**蒸**」这个字——**蒸馏**的蒸。

**蒸笼做的事**：从聊天记录、公开讲演、访谈、文章里，把一个人**说话做事想问题**的方式，蒸成一份 AI 能加载的 Skill。不是复制粘贴原文，是**结构化地理解 TA 怎么想、怎么讲、怎么做决定**。

蒸笼**不是**另一套引擎——它和仓库根目录的 **[永生蒸馏器](../SKILL.md)** 共用同一套采集、提取、封包工具。差别只有一个：**蒸笼更好讲，更适合给第一次听「蒸馏」的人解释。**

## 为啥能蒸名人

因为 TA 们最值钱的东西，**大量公开**：讲演、长文、访谈、播客、开源社区发言、公开课。这些是 **可追溯、合法、不编隐私** 的公开资料。

蒸笼帮你做的是：

- **不是** 让真人来给你打工，**不是** 冒充本人，**不是** 编造 TA 没说过的话。
- **是**：把 TA 公开分享过的**知识框架、决策逻辑、表达习惯**，结构化成一份你的 AI 能读的参谋——**你仍然是拍板的人**。

> 你花了那么多时间追 TA 的内容、买 TA 的课、看 TA 的直播——
>
> 让这些公开方法论在你的对话框里**随叫随到**，不过分吧？

**边界写死了**：只用**公开、可追溯**的出处；不碰未授权隐私；不用于欺骗或骚扰。具体规则见 [`../personas/public-figure.md`](../personas/public-figure.md)。

## 蒸出来长啥样

和主仓库一致：一个人 = 一个文件夹 = 一套可加载的 Skill 文件。

```
elon-public/
├── SKILL.md          ← AI 读这个就知道「TA 怎么想」
├── interaction.md    ← TA 公开场合怎么说话、怎么回应
├── procedure.md      ← TA 怎么做决策、怎么拆问题
├── personality.md    ← 公开形象里展现的性格特征
├── memory.md         ← TA 公开讲过的经历和转折点
├── conflicts.md      ← 不同来源说法不一致的地方
└── manifest.json     ← 元数据
```

丢进你的 Agent Skills 目录，AI 就能用 TA 公开展现过的思路帮你想问题了。
