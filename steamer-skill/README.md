## 这句人话先说清

**蒸笼**不是玄学，就是厨房那层竹笼、不锈钢蒸格——**蒸馒头、包子、早点**，谁没见过？

我们把 **「蒸」** 和 **「蒸馏」** 接上线：把一个人**说话、做事、想问题的方式**，从聊天记录、文档、公开讲演里，**蒸**成一份能加载进 Agent 的 Skill——**不是复制粘贴，是结构化理解**。

## 蒸馏任何人

> 同事、前任、奶奶、你自己——**任何人**。  
> 本入口与仓库根 **[`SKILL.md`](../SKILL.md)** 共用**同一套引擎**；蒸笼只负责**更好讲、更好记**。

| 你大概想…… | 蒸笼怎么说 | 引擎里对应啥 |
|------------|------------|----------------|
| 蒸一个「像 TA 一样想问题」的人 | 掀开蒸笼，取一份「TA 的脑回路」 | 多角色模板 + 四维蒸馏 |
| 蒸公开大佬当参谋 | 借公开讲演里的方法论，不当狗仔 | `personas/public-figure` + 仅公开出处 |
| 先讲给爸妈听 | 蒸馒头 = 蒸馏，一句就懂 | 叙事入口；技术细节仍看根 `SKILL.md` |

## 蒸馏任何人——包括你追过的「顶流」

**任何人**里，单独拎一类：**公众人物 / 大佬 / 顶流**。

他们最值钱的往往不是八卦，而是 **可迁移的知识框架**：怎么决策、怎么表达、怎么拆问题。这些东西大量躺在 **公开讲演、长文、访谈、开源社区** 里——**合法、可追溯、不编隐私**。

所以我们会半开玩笑地说：

> **顶流挣了我那么多注意力（和时间），让我在对话框里借一下 TA 公开讲过的「脑回路」，当一回私人参谋，不过分吧？**

**翻译成产品人话：**

- **不是**让真人来给你打工，**不是**冒充本人，**不是**伪造发言。
- **是**：把 **公开方法论** 蒸馏成你本地的**顾问视角**，**你仍是拍板的人**；学习辅助 ≠ 代言。

**边界**：只基于 **可追溯的公开出处**；不碰未授权隐私、不用于欺骗或骚扰。详见 [`../personas/public-figure.md`](../personas/public-figure.md)。

## 怎么开蒸

**方式一：把仓库丢给 AI（推荐）**

1. 克隆本仓库，**工作目录设在仓库根**（`immortal-skill/`）。
2. 打开 **[`../FOR_AI.md`](../FOR_AI.md)**，复制 **第二节（蒸笼）** 或整段「一句话总控」。
3. AI 会读 [`SKILL.md`](SKILL.md)，需要采集时执行 `python3 kit/immortal_cli.py ...`（路径相对仓库根）。

**方式二：自己当人类产品经理**

1. 先读 [`recipes/steamer-flow.md`](recipes/steamer-flow.md) 与根目录 [`../recipes/intake-protocol.md`](../recipes/intake-protocol.md)。
2. 选角色 → 准备材料 → 按根 `SKILL.md` 里 Phase 走。

**方式三：只想看蒸好的长啥样**

直接逛 [`../examples/`](../examples/) 里三个 demo——结构和蒸笼产出一致。
